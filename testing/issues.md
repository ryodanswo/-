# Список обнаруженных проблем (Issues)

## Баги (Ошибки кода)
Тест ID 3 (Failed) показал ошибку логики валидации (использовался оператор &&). После исправления проверка пустых полей выполняется корректно.

Тест ID 7 (Warning) выявил UX-проблему: при отмене редактирования исходная строка уже удалена. Рекомендуется менять запись только после подтверждения сохранения изменений.

## Основные пожелания

- Добавить кнопку **«Очистить»**, чтобы быстро стереть введённые данные, если пользователь передумал добавлять товар.
- При редактировании товара сделать так, чтобы было более явно, что изменения сохраняются (например, менять надпись кнопки на «Сохранить изменения»).
- Добавить подтверждение перед удалением товара, чтобы защититься от случайных нажатий.
- Реализовать выбор срока годности через календарь (`DatePicker`), а не вводить дату вручную.
- Сделать сохранение списка товаров в файл, чтобы при повторном запуске приложения данные не пропадали.
- В будущем — добавить поле для поиска/фильтрации товаров по названию или категории.
