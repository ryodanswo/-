# План тестирования (Test Plan)

---

## 1. Введение

Этот документ описывает стратегию тестирования приложения **«Складской учет товаров»**.  
**Цель:** убедиться, что приложение корректно добавляет, редактирует и удаляет товары, правильно обрабатывает операции приёма и выдачи, валидирует вводимые данные и корректно обновляет статус товара («В наличии» / «Нет в наличии»).

---

## 2. Объект тестирования

Приложение `WarehouseApp.java`. Функционал:

1. Валидация полей ввода:
   - Название товара;
   - Категория;
   - Поставщик;
   - Количество;
   - Срок годности (гггг-мм-дд).

2. Операции работы со списком товаров:
   1. Создание (добавление) нового товара.
   2. Редактирование существующего товара.
   3. Удаление выбранного товара.
   4. Просмотр списка товаров в таблице `TableView<Product>`.

3. Складские операции:
   1. Приём товара (увеличение количества).
   2. Выдача товара (уменьшение количества).
   3. Контроль недостаточного количества при выдаче.

4. Логика статуса товара:
   - При количестве `> 0` статус — **«В наличии»**.
   - При количестве `= 0` статус — **«Нет в наличии»**.
   - Автоматическое обновление статуса при изменении количества (через кнопки и редактирование в таблице).

---

## 3. Инструменты

- **Среда выполнения:** JavaFX.
- **Метод:** ручное функциональное тестирование (позитивные и негативные сценарии).
- **Окружение:** Windows 10/11, JDK 17+.
- **Дополнительно (опционально):**
  - JUnit 5 для модульного тестирования класса `Product`.
  - Таблица тест-кейсов и отчёты в папке `testing/`.

---

## 4. Стратегия

Будут проведены:

1. **Позитивные тесты** (корректные данные):
   - добавление товара с валидными значениям всех полей;
   - редактирование существующей записи;
   - корректное удаление выбранного товара;
   - приём и выдача товара в пределах доступного количества;
   - проверка смены статуса при количестве 0 и больше 0.

2. **Негативные тесты** (ошибочные данные и действия):
   - попытка добавить товар при пустых полях;
   - ввод букв и некорректных символов вместо числового значения в поле «Количество» и в диалогах;
   - попытка выдать больше товара, чем есть на складе;
   - проверка реакции приложения на неверный ввод количества (сообщения об ошибке).
